# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in TicketKit, please report it by emailing **owais.munawar@gmail.com or dai@kiara.team** or by opening a private security advisory on GitHub.

**Please do not report security vulnerabilities through public GitHub issues.**

### What to Include

When reporting a vulnerability, please include:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if any)

### Response Timeline

- We will acknowledge your report within 48 hours
- We will provide an initial assessment within 7 days
- We will work with you to understand and resolve the issue
- We will notify you when the vulnerability is fixed

## Security Best Practices

When using TicketKit:

- Keep dependencies updated
- Use file-based storage with appropriate file permissions
- Validate and sanitize all user input
- Implement proper authentication and authorization
- Use HTTPS for all API communications
- Regular security audits of your implementation

## Disclosure Policy

- Security vulnerabilities will be fixed in the next patch release
- Credit will be given to security researchers who responsibly disclose issues
- Public disclosure will be coordinated with the reporter
